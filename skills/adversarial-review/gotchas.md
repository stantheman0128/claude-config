# Gotchas — 辯論審閱常見問題

## 致命級

### 1. 使用 Subagents 而非 Agent Teams
- **問題**：用 Subagent 產出各自獨立的審閱，沒有真正的對話碰撞
- **正確做法**：必須使用 Agent Teams，Teammates 直接互相 message

### 2. 回合制結構取代自然對話
- **問題**：設計 Round 1、Round 2、Round 3，每個角色輪流發言
- **正確做法**：讓 Teammates 自由交流，像三個人在會議室聊天。看到對方說了什麼就回應

### 3. 三個角色說同樣的話
- **問題**：三位評審的觀點高度重疊，沒有真正的維度衝突
- **正確做法**：seed questions 要刻意製造框架碰撞。如果三人都同意，追加 follow-up 逼出分歧

## 重大級

### 4. Seed Questions 品質差
- **問題**：問題太表面（「報告寫得好嗎？」）或可以用是/否回答
- **正確做法**：每個問題都要逼出觀點。聚焦報告的沉默區域和隱含假設

### 5. Lead 過度介入
- **問題**：Lead 每隔幾句就 broadcast 指導，打斷討論節奏
- **正確做法**：Lead 只在停滯、循環、收斂三種情況介入。大部分時間靜觀

### 6. 黑臉的事實指控未驗證
- **問題**：黑臉說「這個數字是錯的」就直接採信，沒有用 Web Search 核查
- **正確做法**：所有事實性指控在納入最終輸出前必須交叉驗證

### 7. 只有防守沒有進攻
- **問題**：討論只找問題，沒有碰撞出新洞見
- **正確做法**：進攻面（second-order insights）是審閱的核心價值。只有透過碰撞才浮現的洞見才值得記錄

## 中度級

### 8. 發言超過 150 字限制
- **問題**：Teammates 寫長篇大論，失去對話節奏
- **正確做法**：嚴格執行 150 字限制。精煉有力，不寫長文

### 9. 立場變化沒有記錄
- **問題**：有人被說服但沒有明確標記，最終整合時遺漏
- **正確做法**：指示 Teammates「被說服時明確說『這點我認』」，Lead 整合時特別收集

### 10. 整合輸出缺少「逆向劇本」
- **問題**：只列出問題和洞見，沒有寫反向投資備忘錄
- **正確做法**：逆向劇本是報告韌性測試。如果有聰明對手要反駁核心論點，他的故事是什麼？
